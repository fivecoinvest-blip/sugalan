## Image Usage

### In Database
When syncing games, store the image filename in the `icon_url` or `image_url` column:

```php
'icon_url' => 'jili_image/code_49 Super Ace.png'
```

### In Frontend
Display game images using the stored path:

```vue
<template>
  <img :src="`/storage/docs/${game.icon_url}`" :alt="game.name" />
</template>
```

### Storage Setup
Create symbolic link to make images accessible:

```bash
# Link docs folder to public storage
ln -s /home/neng/Desktop/sugalan/docs/jili_image /home/neng/Desktop/sugalan/public/storage/jili_image
```

Or copy images to public directory:

```bash
cp -r docs/jili_image public/images/games/jili
```

Use this PHP script to bulk import JILI games with images:

```php
<?php
// Import JILI games with images

$gamesData = [
    ['code' => 2, 'name' => 'Chin Shi Huang', 'uid' => '24da72b49b0dd0e5cbef9579d09d8981', 'image' => 'code_2 Chin Shi Huang.png'],
    ['code' => 4, 'name' => 'God Of Martial', 'uid' => '21ef8a7ddd39836979170a2e7584e333', 'image' => 'code_4 God Of Martial.png'],
    // ... add all games
];

$providerId = 1; // JILI provider ID
$category = 'slots';

foreach ($gamesData as $data) {
    DB::table('slot_games')->updateOrInsert(
        [
            'provider_id' => $providerId,
            'game_id' => $data['uid']
        ],
        [
            'name' => $data['name'],
            'category' => $category,
            'icon_url' => 'jili_image/' . $data['image'],
            'is_active' => true,
            'updated_at' => now(),
            'created_at' => now(),
        ]
    );
}
```
