Quick Start

Quick Start Guide

Get started in 3 easy steps

This API is used to start a new game for a user.

1

Launch Game API ‚Äî Start a Game Session

Launch game session with encrypted user data

Imagine you own a gaming website called PlayZone. A user named Jhon logs into your site and clicks on ‚ÄúPlay Sweet Magic‚Äù. Now your website needs to open that game from another company ‚Äî let‚Äôs say the game provider is JILI.
But the provider won‚Äôt just open the game directly. They need some important information first ‚Äî who is playing, how much balance they have, and what game they want to play.
So, your website sends a secure request to the SoftAPI server using their Launch Game API.

üß© What You Send in the Request:


Parameter Meaning Example
user_id The unique ID of the player 101
balance The player's wallet balance 500
game_uid A unique ID for this game session 784512
token Your API token (provided by SoftAPI) 3dfu98r2q3x
timestamp Current time (in milliseconds) 1696329392000
return URL where the user returns after playing https://playzone.com/return
callback URL where the game sends result https://playzone.com/callback.php
currency_code (Optional) Currency code for the game (e.g., BDT, USD, INR) BDT
language (Optional) Language code for the game interface (e.g., bn, en, hi) bn


Once your site collects these details, it encrypts them using a special PHP function called ENCRYPT_PAYLOAD_ECB(). This turns your readable information (like user ID and balance) into a secret code that only SoftAPI can read. You then send this encrypted data to the SoftAPI launch endpoint ‚Äî https://igamingapis.live/api/v1 ‚Äî along with your token. The API checks the data, decrypts it using your secret key, and then returns a game URL in the response.

That URL looks something like this:
https://api.igamingapi.online/launch?de=8787e8d09c5bc0c&game_name=Sweet+Magic&category=flash&brand_title=JILI

Your website simply redirects the player to that URL ‚Äî and the game opens instantly in the browser!

In short, your website prepares the player info ‚Üí encrypts it for safety ‚Üí sends it to SoftAPI ‚Üí gets back a unique launch link ‚Üí and uses that to start the game securely and instantly.

PHP Implementation 
<?php
// === Example: Launch Game via SoftAPI ===

// Step 1Ô∏è‚É£: Set your API credentials (from your SoftAPI account)
$TOKEN  = "3dfu98r2q3x";  // Your unique API token
$SECRET = "12345678901234567890123456789012"; // 32-character secret key

// Step 2Ô∏è‚É£: Game details & player info
$SERVER_URL   = "https://igamingapis.live/api/v1"; // API endpoint
$RETURN_URL   = "https://playzone.com/return";       // After game ends
$CALLBACK_URL = "https://playzone.com/callback.php"; // Game result callback
$USER_ID      = 101;                                // Player ID
$BALANCE      = 500;                                // Player wallet balance
$GAME_UID     = 784512;                             // Unique game session ID

// Step 3Ô∏è‚É£: Encrypt data using AES-256-ECB
function ENCRYPT_PAYLOAD_ECB(array $DATA, string $KEY): string {
    if (strlen($KEY) !== 32) throw new Exception("Key must be 32 bytes long");
    $JSON = json_encode($DATA, JSON_UNESCAPED_UNICODE);
    $ENC  = openssl_encrypt($JSON, "AES-256-ECB", $KEY, OPENSSL_RAW_DATA);
    return base64_encode($ENC);
}

// Step 4Ô∏è‚É£: Prepare the data payload
$PAYLOAD = [
    "user_id"   => $USER_ID,
    "balance"   => $BALANCE,
    "game_uid"  => $GAME_UID,
    "token"     => $TOKEN,
    "timestamp" => round(microtime(true) * 1000), // current time in ms
    "return"    => $RETURN_URL,
    "callback"  => $CALLBACK_URL
    // Optional parameters (can be omitted):
    // "currency_code" => "BDT",  // Optional: Currency code
    // "language" => "bn"         // Optional: Language code
];

// Step 5Ô∏è‚É£: Encrypt it
$ENCRYPTED = ENCRYPT_PAYLOAD_ECB($PAYLOAD, $SECRET);

// Step 6Ô∏è‚É£: Send it to the SoftAPI server
$URL = $SERVER_URL . "?payload=" . urlencode($ENCRYPTED) . "&token=" . urlencode($TOKEN);

$CH = curl_init($URL);
curl_setopt($CH, CURLOPT_RETURNTRANSFER, true);
$RESPONSE = curl_exec($CH);
curl_close($CH);

// Step 7Ô∏è‚É£: Decode the response
$DATA = json_decode($RESPONSE, true);

// Step 8Ô∏è‚É£: Show the result
if ($DATA["code"] == 0) {
    echo "‚úÖ Game launched successfully!<br>";
    echo "üéÆ Game URL: " . $DATA["data"]["url"];
} else {
    echo "‚ùå Error: " . $DATA["msg"];
}
?>
2

Receive Game Callback

Process game results from our server

After a player finishes a game, the game provider (SoftAPI / JILI) automatically sends a callback to your server to report the results of the game session. For example, imagine a user named Jhon plays ‚ÄúSweet Magic‚Äù on your website PlayZone. He bets ‚Çπ50 and wins ‚Çπ30. The game server sends a secure message to your callback.php script with details like the game session ID, round ID, player account, bet amount, win amount, and timestamp. Your server receives this data, calculates the net credit (bet minus win), updates the user‚Äôs wallet accordingly, and logs the result in the database. It then responds back to the game server with a JSON object containing the credit_amount and timestamp. This automatic callback process ensures that the player‚Äôs balance is always accurate, the game results are secure and tamper-proof, and no manual intervention is needed. Essentially, your site acts like a real-time referee: it receives the game outcome, verifies it, updates the player‚Äôs account, and confirms back to the provider ‚Äî keeping the whole process smooth, secure, and reliable.

    "game_uid": "3978",

    "member_account": "23213",
    "bet_amount": 1,
    "win_amount": 0,
    "timestamp": "2025-10-14 16:41:45"
}

Update user balance and save transaction

The game server sends a callback after each bet or win with details like user ID, bet amount, win amount, and game round. Your system receives this data, updates the player's wallet (subtract bet, add win), and logs the bet in your database for history and tracking.

Launch Game

Launch Game API

Start a game session for your users
GET
https://igamingapis.live/api/v1

This API is used to start a game session for a user. You send user info, balance, and session ID to the server, and it returns a URL to launch the game.

Note: The parameters currency_code and language are optional. Your code will work perfectly fine without passing these parameters. You can omit them if you don't need to specify currency or language.
